# 우선순위 정렬 기능 — 구현 문서

> 피해 규모 금액 및 피해자 수 기반으로 사건을 자동 정렬하여 검토 우선순위를 추천하는 기능
> 최종 수정: 2026-02-27

---

## 왜 이 기능이 필요한가?

기존에는 분석 목록이 **"최신 분석순"** 으로만 정렬되었습니다.
실무에서는 "피해 규모가 큰 사건부터 검토하고 싶다"는 니즈가 있지만,
LLM이 반환하는 `damage_amount`, `victim_count` 값이 **자연어 텍스트**여서 숫자 정렬이 불가능했습니다.

```
기존 LLM 출력 예시
─────────────────
damage_amount: "약 500억원"      ← 문자열
victim_count:  "1만 2천명 이상"   ← 문자열
               "미상"             ← 파싱 불가
```

이 기능은 텍스트를 숫자로 파싱하여 DB에 저장하고, 빠른 정렬을 가능하게 합니다.

```

┌──────────────────────┬────────────┬────────────────────────┐

├──────────────────────┼────────────┼────────────────────────┤
```

### 2. 파싱 함수 (`backend/analyses/tasks.py`)

#### parse_damage_amount() — 금액 텍스트 → 원(KRW) 정수

```
입력 텍스트            파싱 결과 (원 단위)        처리 방식
─────────────────────  ───────────────────────    ─────────────────
"약 500억원"         → 50,000,000,000            500 × 1억
"1,200만원"          → 12,000,000                1,200 × 1만
"2조 3천억원"        → 2,300,000,000,000         2조 + 3천억
"50억원"             → 5,000,000,000             50 × 1억
"5천만원"            → 50,000,000                5천 × 1만
"미상"               → null                      파싱 불가
"수백억 원대"        → null                      숫자 추출 불가
```

**단위 기준표:**

| 한국어 단위 | 숫자 값 |
|------------|---------|
| 조 | 1,000,000,000,000 |
| 억 | 100,000,000 |
| 천만 | 10,000,000 |
| 만 | 10,000 |
| 원 | 1 |

#### parse_victim_count() — 피해자 수 텍스트 → 정수

```
입력 텍스트            파싱 결과 (명)    처리 방식
─────────────────────  ──────────────    ─────────────────
"약 1만 2천명"       → 12,000           1만 + 2천
"50명 이상"          → 50               순수 숫자 추출
"약 3만명"           → 30,000           3 × 1만
"1천 5백명"          → 1,500            1천 + 5백
"미상"               → null             파싱 불가
```

#### 파싱 로직 흐름도

```
텍스트 입력
    │
    ▼
미상/불명/알 수 없음 포함?  ─── Yes ──→ null 반환
    │
    No
    ▼
숫자(\d)가 하나라도 있는가?  ─── No ───→ null 반환
    │
    Yes
    ▼
조 단위 추출 → 억 단위 추출 → 만 단위 추출 → 원 단위 추출
    │
    ▼
합산 결과가 0인가?  ─── Yes ──→ null 반환
    │
    No
    ▼
정수 반환 (원 단위)
```

## 사용 방법

### 프론트엔드에서 사용

1. `http://localhost:5173/analyses` 접속
2. 상단 필터 바에서 **"정렬"** 드롭다운 클릭
3. **"피해 규모 큰 순"** 또는 **"피해자 많은 순"** 선택
4. 테이블이 자동으로 재정렬됨

### API로 직접 호출

```bash
# 피해 규모 큰 순 (Top 10)
GET /api/analyses/?ordering=-damage_amount_num&page=1

# 피해자 많은 순 + High 적합도 필터
GET /api/analyses/?ordering=-victim_count_num&suitability=High

# 피해 규모 큰 순 + 사건별 묶기
GET /api/analyses/?ordering=-damage_amount_num&group_by_case=true
```

## 한계 및 주의사항

### 파싱 실패 케이스 (null 처리)

| 입력 예시 | 이유 | 처리 |
|----------|------|------|
| `"미상"` | 수치 없음 | null |
| `"수백억 원대"` | 숫자 없음 | null |
| `"수십만명"` | 숫자 없음 | null |
| `"약 500만~1천만원"` | 범위 표현 | 500만으로 파싱 (하한값) |
| `"$500만"` | 달러 단위 | null (원화 기준) |

### 기존 데이터 파싱률

- 현재 기존 데이터의 **약 15~26%** 만 파싱 성공
- 나머지는 LLM이 "미상"으로 답변한 경우
- **신규 분석 기사**는 LLM이 수치를 추출할 때마다 자동으로 저장됨

### 재파싱 방법

만약 파싱 로직을 개선한 뒤 기존 데이터를 다시 파싱하려면:

```bash
# Django shell에서 실행
just shell

>>> from analyses.tasks import parse_damage_amount, parse_victim_count
>>> from analyses.models import Analysis
>>> to_update = []
>>> for a in Analysis.objects.all():
...     a.damage_amount_num = parse_damage_amount(a.damage_amount)
...     a.victim_count_num = parse_victim_count(a.victim_count)
...     to_update.append(a)
>>> Analysis.objects.bulk_update(to_update, ['damage_amount_num', 'victim_count_num'])
```
